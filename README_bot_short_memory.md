# bot_short_memory.py — Telegram-бот с короткой памятью

Telegram-бот на **aiogram 3.x** с поддержкой нескольких AI-провайдеров и буфером истории диалога в оперативной памяти.

---

## Быстрый старт

### 1. Создать и активировать виртуальное окружение

**Windows:**
```bash
python -m venv venv
venv\Scripts\activate
```

**Linux / macOS:**
```bash
python -m venv venv
source venv/bin/activate
```

### 2. Установить зависимости

```bash
pip install -r requirements.txt
```

### 3. Запустить бота

```bash
python bot_short_memory.py
```

Убедись, что файл `.env` содержит нужные ключи (см. раздел «Конфигурация»).

---

## Конфигурация (.env)

```env
BOT_TOKEN=<токен Telegram-бота от @BotFather>

ZAI_API_KEY=<ключ Z.AI>
PROXY_API_KEY=<ключ ProxyAPI>
GEN_API_KEY=<ключ GenAPI>
```

## Архитектура

```
bot_short_memory.py
│
├── PROVIDERS dict       — конфигурация провайдеров и моделей
├── memory dict          — короткая память: user_id → deque(maxlen=10)
├── user_settings dict   — настройки: user_id → {provider, model, temp, tokens}
│
├── ConfigStates (FSM)   — состояния для пошаговой настройки /config
│
├── Handlers
│   ├── /start, /info, /new, /config   — команды
│   ├── callback_query prov:/model:/tokens:  — inline-кнопки конфигурации
│   └── F.text           — основной обработчик сообщений
│
└── main()               — запуск бота (long polling)
```

### Поток обработки сообщения

```
Пользователь пишет текст
        │
        ▼
Добавить в memory[user_id]  (deque maxlen=10)
        │
        ▼
Собрать payload: [system_msg] + list(memory[user_id])
        │
        ▼
OpenAI(api_key, base_url).chat.completions.create(...)  ← asyncio.to_thread
        │
        ▼
Отправить ответ пользователю
        │
        ▼
Добавить ответ ассистента в memory[user_id]
```

## Параметры настройки (/config)

Настройка проходит в 3 шага через inline-кнопки и FSM:

1. **Провайдер** — выбор из трёх кнопок
2. **Модель** — список моделей выбранного провайдера
3. **Температура** — вводится текстом в диапазоне, допустимом для модели
4. **Макс. токены** — выбор из кнопок или ввод вручную

Настройки хранятся в `user_settings` (оперативная память, сбрасываются при перезапуске).

---

## Короткая память

- Хранится в `collections.deque(maxlen=10)` для каждого `user_id`
- Содержит чередующиеся сообщения `user` и `assistant`
- При каждом запросе к модели добавляется системный промпт в начало
- `/new` полностью очищает буфер пользователя
